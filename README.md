## ◈ Featured Projects

<details>
<summary><b>&nbsp;⬡&nbsp; LabelFlow — AI Data Annotation Platform</b></summary>

<br/>

> A full-stack, web-based data annotation platform purpose-built to accelerate machine learning workflows. LabelFlow enables administrators to manage the complete lifecycle of image dataset labeling — from project creation and dataset ingestion to task assignment, quality review, and export — all within a single, unified interface.

<div align="center">

| Attribute | Detail |
|---|---|
| **Stack** | Python · JavaScript · HTML · CSS · Django · Django REST Framework |
| **Architecture** | MVC · REST API-first · Role-based Access Control |
| **Core Features** | Project Management · Image Upload · Annotation Task Assignment · Submission Review · Dataset Export |
| **ML Integration** | Export labeled datasets in training-ready formats for downstream ML pipelines |
| **Security** | Admin / Annotator role separation · Authenticated task access |
| **Impact** | Eliminates dependency on third-party annotation tools · Enables fully self-hosted ML data pipelines |
| **Repository** | [![GitHub](https://img.shields.io/badge/View%20Repository-181717?style=flat-square&logo=github)](https://github.com/kirandahal63/LabelFlow) |

</div>

LabelFlow was designed to address a genuine pain point in the ML development lifecycle: the absence of an accessible, self-hosted annotation platform for teams that cannot afford or trust commercial alternatives. The system implements a clean separation between administrator and annotator roles, ensuring data governance at every stage. Image datasets are uploaded, chunked into tasks, assigned to annotators, reviewed against quality standards, and finally exported as structured JSON/CSV datasets ready for training.

<br/>
</details>

<summary><b>&nbsp;⬡&nbsp; Gantavya — Intercity Travel Booking & Management System</b></summary>

<br/>

> A centralized, full-stack web application engineered to modernize Nepal's fragmented and manual intercity travel ticketing ecosystem. Gantavya consolidates booking, scheduling, and management operations into a single, reliable platform — replacing paper-based processes with a digital-first experience for both operators and passengers.

<div align="center">

| Attribute | Detail |
|---|---|
| **Stack** | Java · Apache Tomcat · XAMPP · phpMyAdmin · HTML · CSS · JavaScript |
| **Architecture** | MVC · Servlet-based Backend · Relational Database |
| **Scale** | Multi-route · Multi-operator · Centralized Booking Engine |
| **Testing** | Manual end-to-end testing of core booking and reservation flows |
| **Security** | Session management · Role-based operator and passenger access |
| **Impact** | Addresses Nepal's underserved intercity travel market · Reduces manual ticketing errors |
| **Repository** | [![GitHub](https://img.shields.io/badge/View%20Repository-181717?style=flat-square&logo=github)](https://github.com/kirandahal63/Gantavya) |

</div>

Gantavya was conceived as a direct response to the inefficiencies plaguing Nepal's intercity travel industry — a sector where ticket booking is still largely handled through physical counters, prone to double-booking, lack of transparency, and poor customer experience. The platform provides travel operators with a management dashboard to configure routes, schedules, and seat availability while offering passengers a streamlined booking interface. Manual testing was conducted rigorously across all critical booking flows to validate correctness and stability prior to delivery.

<br/>
